如何做功能点的调研

什么情况下要做功能的调研？竞品出了新功能、自身产品功能需要改进或者自身产品准备上新功能、自我驱动学习

一般对于类似支付宝等大型复杂的App做功能调研，对于简单小型的App做产品调研

功能调研和产品调研有什么区别？

功能点调研怎么做？

  1. 明确调研的目的

  2. 如果调研类型是：竞品出了新功能

    想想核心问题：都有谁会用？关键参与者都是谁？用户为什么要用这个？有什么好处？分别都是怎么用的？

    查看数据表现：覆盖率、发布量、成交量等

    对于自己的产品来说，有什么意义？

  3. 如果调研类型是：自身产品功能需要改进或者自身产品准备上新功能

    想想核心问题：你的目标用户是谁？先思考再调研；产品功能的现状是如何的？先知己再带着目的调研；调研的目标产品有哪些？找行业领先、产品口碑领先；功能的核心关键点是什么？功能逻辑是什么？功能调研的关键；差异点是什么？选择和做决策

  4. 如果调研类型是：自我驱动学习

    想想核心问题：

功能调研的误区：没结论、不愿意做、杂、非要提改进意见

针对独立产品的调研怎么做？

  1. 用户、场景、需求是怎么被满足的：用户（有哪些类型的用户）、场景（不同类型的用户分别在什么场景下会使用）、需求（产品满足了用户的哪些需求）
  2. 产品逻辑：用户的流向（都有哪些用户，怎么发生联系的）、数据的流向（产生的数据流是如何运转的）、汇合点（用户、数据在哪个关键页面汇合）、关键路径（用户操作的关键功能、页面是什么）
  3. 关键页面与典型交互
  4. 迭代路径
  5. 产品的表现如何：数据表现（使用数据、排名数据、评分数据）、用户表现（用户点评、产品点评）
  6. 运营路径与方法

多个同类产品怎么调研？

  1. 找相同，找不同
  2. 关键功能的差异
  3. 出结论：A和B相比，优点、缺点、亮点；如果要抄，抄哪个？抄什么？怎么抄？

如何调研第三方产品是否能用？

明确目的比功能很全面更重要

  1. 目标：用来做什么？期望值？如客服系统、直播分享等
  2. 成本：如果自己做，从能用到好用是如何的？
  3. 关联性：第三方产品和你完成目标之间的关联
  4. 其他：价格、服务、会不会倒闭等

产品调研的误区

  1. 产品调研不等于行业调研
  2. 二三手的数据不能证明你正确
  3. 非得加改进建议
  4. 大而全（要做目标明确、直指要点、详略得当的调研）
  5. 强烈的个人倾向

产品调研的总结

为什么要做调研？什么时候需要做产品调研？

  1. 提高产品感：见过、用过、思考过；理解行业新趋势，懂得如何运用；同行交流的谈资
  2. 提升工作效率：追踪竞品，知道背后的意义，跟不跟？怎么跟？同业异业合作，提前知己知彼不做无用功；抄是一门艺术，而不仅仅是一门技术
